1. 디바이스 정책
1.1. os
해당 시스템의 경우 Windows (특히, Windows 10/11 Pro 이상), MacOS (최신 버전), 
Linux (Ubuntu, Red Hat, CentOS 등 주요 배포판) 만을 허용하고 있음
Windows XP, Windows Vista, Windows 7는 구식 브라우저의 보안 이슈로 거부하며,
Android, iOS 등의 모바일용 os는 업무용 시스템에 부적합함.
Unix, OpenBSD, Chronium OS 등의 특수 os는 기업에서 잘 사용하지 않음

1.2. 브라우저
해당 시스템의 경우 Google Chrome, Microsoft Edge, Mozilla Firefox, Apple Safari, 
Opera, Brave, Vivaldi, Yandex Browser의 8개 브라우저 정보를 수집할 수 있으며, 
이중 Chrome, Edge, Firefox, Safari, Brave만을 허용하고 있음.

거부 브라우저들의 거부사유는 아래와 같음.
Yandex Browser: 법적/정치적 이유(러시아 브라우저)로 일부 국가에서는 사용을 제한하거나 거부할 수 있음.
Opera: 일부 기업 환경에서는 보안이 강화된 Chrome 또는 Edge를 선호하며, 호환성 문제가 있을 수 있음.
Vivaldi: 개인화가 과도한 설정이 많아서 보안과 관리 측면에서 다소 불안할 수 있음.

이외 식별이 불가한 브라우저는 허용하지 않음.

1.3. 네트워크
networkInfo로는 isOnline, networkType, downlink, rtt, saveData이 전송됨.
이 중 isOnline은 웹 프로그램으로서 검증이 필요없고, networkType의 경우
블루투스, 셀룰러, 이더넷, none, mixed, other, unknown, wifi, wimax으로
네트워크 타입을 분류함.

이 중 이더넷 (ethernet), Wi-Fi (wifi), WiMAX (wimax)만을 허용함.
블루투스는 데이터 전송속도가 느리고 보안성이 낮으며(다수의 보안이슈), 
셀룰러는 공용망이나 중계기 문제로 신뢰할 수 없으며, other, unknown, none은 허용불가

downlink는 웹 페이지 로딩, API 요청 등과 같이 시스템의 원활한 접속을 위해
최소 3 Mbps 이상을 요구함

rtt는 원활한 화상회의나 실시간 협업을 위해, 일반 기업의 요구치인 250ms 이하를 요구함

SaveData(데이터 절약모드)는 웹 트래픽을 압축하거나 제한하여 시스템 정상 작동을 방해할 수 있음.
따라서 VPN, 보안 통신이 제대로 동작하지 않을 가능성이 있으므로, 비활성화할 경우에만 허용함
